# FaceCheck 本地开发启动指南

> 适用环境：Windows + Docker Desktop + Android Studio + Flutter  
> 最后更新：2026-06-01

---

## 前置条件

| 工具 | 说明 |
|------|------|
| Docker Desktop | 运行数据库/缓存/消息队列 |
| Android Studio | 运行 Android 模拟器 |
| Flutter SDK | 构建并运行 App（本项目使用 `D:\flutter`） |
| JDK 22 | 后端编译运行（Android Studio 自带 JBR 即可） |

---

## 一、启动基础设施（Docker）

打开 **PowerShell**，进入项目根目录：

```powershell
cd D:\projects\FaceCheck
docker compose up -d postgres redis rabbitmq
```

验证三个容器都在运行：

```powershell
docker compose ps
```

应该看到 `facecheck-postgres`、`facecheck-redis`、`facecheck-rabbitmq` 都是 `Up` 状态。

> ⚠️ 注意：本项目 Redis 映射到了宿主机的 **26379** 端口（而非默认的 6379），因为 6379 被 Windows Hyper-V 保留了。

---

## 二、启动后端（Spring Boot）

在 **新的 PowerShell 窗口** 中执行：

```powershell
cd D:\projects\FaceCheck\backend

$env:SPRING_PROFILES_ACTIVE = "local"
$env:JWT_SECRET = "change-this-to-a-long-random-local-secret"
$env:HUAWEI_CLOUD_ENABLED = "false"
$env:SPRING_RABBITMQ_LISTENER_SIMPLE_AUTO_STARTUP = "true"

.\mvnw.cmd spring-boot:run
```

看到 `Started FaceCheckApplication` 即启动成功。

验证：

```powershell
Invoke-RestMethod -Uri http://localhost:8080/api/health
```

## 四、启动 Android 模拟器

1. 打开 **Android Studio**
2. 点击 **More Actions → Virtual Device Manager**
3. 找到已创建的设备（如 Pixel 6），点击右侧 **▶ 播放键**
4. 等待模拟器开机进入安卓桌面

---

## 五、启动 Flutter App

在 **新的 PowerShell 窗口** 中执行：

```powershell
cd D:\projects\FaceCheck\app
flutter pub get
flutter run
```

App 会自动安装到已启动的模拟器上。

当前 App 默认连接已部署后端：`https://115.120.241.220`。该入口使用自签名证书，Android 真机或模拟器需要先安装并信任证书。

如果要让模拟器连接本机后端，请显式覆盖后端地址：

```powershell
flutter run `
  --dart-define=FACECHECK_BASE_URL=http://10.0.2.2:8080 `
  --dart-define=FACECHECK_LOCAL_BACKEND_HOSTS=10.0.2.2,127.0.0.1,localhost
```

## 七、常见问题

| 问题 | 解决方法 |
|------|----------|
| 端口 8080 被占用 | `netstat -ano \| findstr 8080` 找到 PID 后 `taskkill /PID xxx /F` |
| Docker 报端口占用 | 检查 `netsh interface ipv4 show excludedportrange`，重启 WinNAT |
| Flutter 报 SDK 版本不兼容 | 检查 `flutter --version`，确保 Dart SDK ≥ 3.4 |
| 模拟器未检测到 | `adb devices` 检查，或重启模拟器 |
| RabbitMQ 连接失败 | 确认 `docker compose ps` 中 rabbitmq 是 Up 状态 |
| Gradle 构建失败：`Minimum supported Gradle version is 8.11.1` | 编辑 `app/android/gradle/wrapper/gradle-wrapper.properties`，将 `distributionUrl` 升级到 `gradle-8.14-all.zip`（AGP 8.7.0 要求 Gradle ≥ 8.11.1） |
| Gradle 构建失败：`NDK did not have a source.properties file` | 打开 Android Studio → SDK Manager → SDK Tools，勾选 **NDK (Side by side)** 安装 |
| Gradle 下载依赖超时 | 在 `app/android/settings.gradle.kts` 和 `app/android/build.gradle.kts` 的 `repositories` 块中，在 `google()` 和 `mavenCentral()` 之前添加 `maven { url = uri("https://maven.aliyun.com/repository/public") }` |
| Gradle 代理问题 | 在 `app/android/gradle.properties` 中配置：`systemProp.http.proxyHost=localhost`、`systemProp.http.proxyPort=7897`（根据实际代理端口调整） |
| Flutter 构建失败：`package_config.json does not exist` | 必须先 `cd app && flutter pub get`，再从 `app` 目录执行 `flutter run`（不要直接从 `android` 目录调用 `gradlew`） |
