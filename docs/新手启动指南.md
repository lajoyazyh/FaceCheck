# FaceCheck 新手启动指南（白话版）

---

### 启动步骤（备忘）

1. __启动 Docker 基础服务__：`docker-compose up -d`
2. __启动后端__：`cd backend && mvnw.cmd spring-boot:run -Dspring-boot.run.profiles=local`
3. __确认模拟器已开机__（Android Studio → AVD Manager → 点击 ▶）
4. __启动 App__：`cd app && flutter run -d emulator-5554`


## 一、这个项目是什么？

一个**人脸签到 App**。

> 管理员在后台创建一个"签到活动"（比如"6月3日下午班会签到"），系统生成一个二维码。
> 参会者用手机扫码 → 拍一张自拍 → 系统自动识别你是谁 → 签到成功。

---

## 二、项目由哪些"零件"组成？

整个项目可以理解为 **7 个零件**，像搭积木一样：

```
┌──────────────────────────────────────────────┐
│  ① Flutter App（你手机上的 App）              │
│     登录、扫码、拍照、看签到记录               │
│     → 它不直接碰数据库或云服务，所有事都通过后端│
└──────────────────┬───────────────────────────┘
                   │ 通过网络发请求（HTTP）
                   ▼
┌──────────────────────────────────────────────┐
│  ② Spring Boot 后端（大脑）                   │
│     接收 App 的请求，判断你是谁、              │
│     照片里是谁、签到是否成功，然后存数据库      │
└──────┬──────────┬──────────┬─────────────────┘
       │          │          │
       ▼          ▼          ▼
┌──────────┐ ┌──────────┐ ┌──────────────┐
│③PostgreSQL│ │④华为云FRS│ │⑤华为云OBS     │
│ (数据库)  │ │(人脸识别) │ │(存照片)       │
└──────────┘ └──────────┘ └──────────────┘
┌──────────┐ ┌──────────┐
│⑥ Redis   │ │⑦RabbitMQ │
│ (临时缓存)│ │(消息队列) │
└──────────┘ └──────────┘
```

| 零件 | 通俗类比 | 干什么的 |
|------|---------|---------|
| **① Flutter App** | 你手机上的 App 界面 | 登录、扫码、拍照、查看签到记录 |
| **② Spring Boot 后端** | 一个始终在运行的"服务员" | 接收 App 请求，处理业务逻辑，操作数据库 |
| **③ PostgreSQL** | 一个超级 Excel 表格 | 存所有数据：用户、签到记录、场次信息 |
| **④ 华为云 FRS** | 一个专门"认脸"的云服务 | 给它照片，它告诉你照片里是谁 |
| **⑤ 华为云 OBS** | 一个网络硬盘 | 存所有照片文件（头像、签到自拍） |
| **⑥ Redis** | 一张超快的便签纸 | 临时记东西：Token 黑名单、防重复提交 |
| **⑦ RabbitMQ** | 一个排队系统 | 照片上传后排队处理，防止瞬间太多请求 |

> **注意**：④ 和 ⑤ 需要华为云账号。如果暂时没有，后端启动时会用"假的"（Mock）来代替，不影响基本功能测试。

## 三、一张图看懂签到全流程

```
管理员创建签到活动 → 生成二维码
         ↓
参会者扫码 → App 确认"这个场次有效"
         ↓
参会者拍自拍 → App 把照片发给后端
         ↓
后端把照片存到 OBS（网络硬盘）
         ↓
后端把照片发给 FRS（认脸服务）→ FRS 返回"最像谁"
         ↓
后端判断：这个人签过到了吗？
    → 没签过：写入数据库一条签到记录 ✅
    → 签过了：返回"已签到，别重复了" ❌
         ↓
后端返回结果给 App → App 显示"签到成功！"
```

## 四、启动前的准备

### 你的电脑上需要装这些：

| 软件 | 怎么检查装好了没 | 如果没装 |
|------|-----------------|---------|
| **Java 17** | 打开终端输入 `java -version` | 去 Oracle 或 Adoptium 官网下载 JDK 17 |
| **Docker Desktop** | 桌面右下角有鲸鱼图标吗？ | 去 docker.com 下载 Docker Desktop |
| **Flutter** | 打开终端输入 `flutter doctor` | 去 flutter.dev 下载 |
| **Android Studio** | 开始菜单能找到吗？ | 去 developer.android.com/studio 下载 |

### Android Studio 里还需要：

1. 打开 Android Studio → 右上角 **Device Manager**（手机图标）
2. 点击 **Create Device** → 选一个手机型号（推荐 Pixel 6）→ 下载系统镜像 → 点 **▶ 播放键** 开机
3. 看到安卓手机桌面出现 = 模拟器准备好了

## 五、启动步骤（按顺序，一步步来）

### 第 1 步：启动基础服务（PostgreSQL + Redis + RabbitMQ）

这三个通过 Docker 一键启动，就像用遥控器一下打开三盏灯。

打开终端，进入项目目录，运行：

```bash
docker-compose up -d
```

**怎么验证成功了？** 运行 `docker ps`，应该看到 3 个容器在运行。

> 如果报错 "docker: command not found"，说明 Docker Desktop 没装或没启动。

---

### 第 2 步：启动后端（Spring Boot）

后端是 Java 程序，用 Maven 启动。

```bash
cd backend
mvnw.cmd spring-boot:run -Dspring-boot.run.profiles=local
```

> 第一次运行会下载很多依赖包，需要几分钟，耐心等待。

**怎么验证成功了？** 看到类似 `Started FaceCheckApplication in X seconds` 就说明好了。

也可以打开浏览器访问 http://localhost:8080/api/health ，看到 `{"status":"UP"}` 就是 OK。

---

### 第 3 步：启动 App（Flutter）

```bash
cd app
flutter run
```

当前 App 默认连接已经部署好的后端：`https://115.120.241.220`。这个线上入口使用自签名证书，手机或模拟器需要先安装并信任证书。

如果你想让模拟器连接自己电脑上刚启动的本地后端，请这样启动：

```bash
cd app
flutter run --dart-define=FACECHECK_BASE_URL=http://10.0.2.2:8080 --dart-define=FACECHECK_LOCAL_BACKEND_HOSTS=10.0.2.2,127.0.0.1,localhost
```

> 第一次运行会下载 Android 构建依赖，也需要几分钟。

**怎么验证成功了？** 模拟器上会出现 FaceCheck App 的登录界面。

## 七、常见报错和解决办法

### 1. "端口被占用"

```
Port 8080 is already in use
```

**原因**：有其他程序占用了 8080 端口。

**解决**：
```bash
# 查谁占了 8080
netstat -ano | findstr :8080
# 关掉那个程序，或者改后端端口
```

### 2. "No devices found"（Flutter 找不到设备）

**原因**：模拟器没开机。

**解决**：去 Android Studio → Device Manager → 点 ▶ 播放键开机。

也可以用命令检查：
```bash
flutter devices    # 看看有没有设备
adb devices        # 看看 ADB 有没有连上
```

### 3. "Docker daemon not running"

**原因**：Docker Desktop 没启动。

**解决**：去开始菜单找到 Docker Desktop，点开它，等鲸鱼图标出现。

### 4. "Could not find com.google.mlkit:face-detection"

**原因**：Gradle 缓存里有旧依赖残留。

**解决**：
```bash
# 清理 Gradle 缓存
rm -rf ~/.gradle/caches
cd app
flutter clean
flutter pub get
flutter run
```

### 5. 模拟器启动了但 App 没反应

**解决**：
```bash
# 重启 ADB
adb kill-server
adb start-server
# 再试
flutter run
```

## 八、下次怎么用（速查版）

以后每次要用，按顺序跑这 3 条命令就行：

```bash
# 1. 启动基础服务（如果 Docker 已经在运行）
docker-compose up -d

# 2. 启动后端
cd backend && mvnw.cmd spring-boot:run -Dspring-boot.run.profiles=local

# 3. 启动 App（新开一个终端窗口）
cd app && flutter run
```

> **关机前记得**：关掉后端（Ctrl+C）、关掉模拟器、Docker 可以不用关。
